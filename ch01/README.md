## DBMS 개요와 MySQL 소개

### DBMS 개요

#### 데이터베이스의 정의와 특징

데이터베이스를 '데이터 집합'이라고 정의하고 DBMS를 'DB를 운영하기 위한 소프트웨어'라고 정의한다. 여기서 많은 사람들이 엑셀도 DB로 착각을 많이 하시는 분들이 있는데 Excel은 DB가 아니다. 왜 아닐까? Excel은 데이터의 집합 단위로 이루어져 있지만 DB는 여러명의 사용자나 응용프로그램이 공유하고 동시에 접근이 가능해야 하지만 Excel은 그렇지 않기 때문이다.

DBMS의 종류에는 MySQL를 말고도 많은 DBMS가 존재하는데, MariaDB, PostgreSQL, Oracle, SQL Server가 존재한다. 또한 DBMS는 몇자기 중요한 특징이 있다.

- 데이터의 무결성: DB안에 데이터는 오류가 있으면 안된다.
- 데이터의 독립성: DB의 저장소를 바꾸고 구성을 바꾼다 해도, 응용프로그램 코드는 변경이 되어서는 안된다.
- 보안: DB안의 데이터는 아무나 접근할 수 없으며, 허가된 사용자만 접근이 가능하다. 또한, 사용자 계정에 따라서 다른 권한을 가져야 한다.
- 데이터 중복의 최소화: 동일한 데이터가 여러 개 중복되어 저장되는 것을 방지한다.
- 응용 프로그램 제작 및 수정이 쉬워짐: 유지보수에 편리해진다.
- 데이터의 안전성 향상: DBMS가 제공하는 백업 및 복원 기능으로 데이터 손실 시, 복구가 가능하다.

#### 데이터베이스의 발전

- 오프라인으로 관리
  - 초기 PC가 없을 시절에는 DB도 당연히 존재하지 않았을 것이며, 그 때 당시에는 오프라인(종이지표)로 직접 수기로 관리하였다.
- 파일시스템의 이용
  - PC의 보급이 이루어지고 나서는 파일 시스템을 이용하여 데이터를 관리하였다. 파일 시스템은 일종의 메모장이라고 생각하면 편리 할 것이다.
    - 하지만, 시간이 지나서 파일의 크기가 커지고 데이터의 양이 많아지면서 데이터의 중복으로 인한 불일치가 발생할 위험이 높아졌다.
- 데이터베이스 관리시스템
  - 파일시스템의 단점을 보완하고 대량의 데이터를 보다 효율적으로 관리하고 운영하기 위해 나타났다.
    - DBMS에 데이터를 구축하고 관리하고 활용하기 위해서 사용되는 언어가 SQL이다.

#### DBMS 분류

DBMS 유형에는 여러가지 유형이 존재한다.

- 계층형 DBMS: 초창기 DBMS
- 망형 DBMS: 계층형 DBMS 문제점을 개선하고 1:1, 1:N, N:M관계가 지원되어 효과적으로 빠른 데이터 추출 가능
- 관계형 DBMS: 데이터베이스는 테이블이라 불리는 최소단위로 구성

#### SQL 개요

SQL은 관계형 DBMS에서 사용되는 언어로, 일반적인 프로그래밍 언어와 다르다. 또한, SQL은 국제 표준화 기관에서 표준화된 내용을 계속 발표하면 업데이트 해나아간다.

- DBMS 제작회사와 독립적이다.
  - SQL은 모든 DBMS 제작회사에 공통적으로 공개되고 각 제작회사는 이 표준 SQL에 맞춰서 DBMS를 개발한다.
- 다른 시스템으로 이식성이 좋다.
- 표준이 계속 발전한다.
- 대화식 언어이다.
- 분산형 클라이언트/서버 구조이다.

### MySQL 소개

MySQL은 Oracle사에 제작한 DBMS이다. 오픈소스로 되어 있어서 누구나 무료로 다운로드해서 사용할 수 있으며, 소스를 제공하기 때문에 소스를 고쳐서 소프트웨어를 더 발전시켜도 된다.

> MySQL은 비상업용이나 교육용으로는 제한없이 사용해도 되지만, 2010년 오라클이 MySQL을 인수하면서 상용으로 사용하기 위해서 상용 라이선스를 취득해야한다.
